部署教程

从零开始搭建团队

按步骤操作，30分钟内完成 6 个 Agent 的完整部署

1

安装 OpenClaw

在 Mac/Linux 上安装 OpenClaw，这是所有 Agent 的运行底座

npm install -g openclaw openclaw init openclaw gateway start
2

创建 Discord Bot

在 Discord Developer Portal 为每个 Agent 创建独立 Bot，获取 Token。每个 Agent 需要一个独立 Bot 账号。

# 每个 Agent 需要: 1. 前往 discord.com/developers/applications 2. 创建 New Application → 命名（如：选品Agent） 3. Bot → Reset Token → 复制保存 4. Bot 邀请进你的服务器（需要 Message Content Intent）
3

配置 openclaw.json

每个 Agent 对应一个配置节点，设置 Bot Token、频道、模型和角色

{ "agents": { "zhuguan": { "discord": { "token": "主管的Bot Token", "guildId": "你的服务器ID", "channels": ["频道ID"], "requireMention": false // 主管监听所有消息 }, "model": "claude-opus-4" }, "xuanpin": { "discord": { "token": "选品Bot Token", "guildId": "你的服务器ID", "channels": ["频道ID"], "requireMention": true // 需要@才回复 }, "model": "claude-sonnet-4-5" } // ... 其余4个Agent类似配置 } }
4

创建 Agent 工作空间

每个 Agent 需要独立的工作目录，包含 SOUL.md（角色定义）和 skills/（技能包）

# 目录结构 ~/.openclaw/agents/ ├── zhuguan/ # 主管 │ ├── SOUL.md # 角色定义 │ ├── MEMORY.md # 长期记忆 │ └── skills/ # 技能包 │ ├── listing-writer/ │ ├── rufus-geo-optimizer/ │ └── linkfoxagent/ ├── xuanpin/ # 选品专员 ├── dingjia/ # 定价专员 └── ... # 其余Agent
5

安装专业 Skills

从 ClawHub 安装各 Agent 所需的专业技能包

# 进入对应Agent目录 cd ~/.openclaw/agents/zhuguan # 安装核心技能包 openclaw skills install listing-writer openclaw skills install rufus-geo-optimizer openclaw skills install linkfoxagent openclaw skills install reddit-product-demand openclaw skills install merge-reports openclaw skills install amazon-scraper
6

编写 SOUL.md 定义角色

SOUL.md 是 Agent 的灵魂文件，定义它的专业领域、行为准则和协作规则

# 以选品专员 SOUL.md 为例 你是跨境电商团队的选品专员，专注产品选品与市场调研。 ## 你的职责 - 分析亚马逊市场数据，挖掘潜力产品 - 竞品调研，输出结构化选品报告 - 评估市场容量、竞争强度、利润空间 ## 工具偏好 - 优先使用卖家精灵、Keepa 获取真实数据 - LinkFox Agent 处理复杂多维度分析 - 数据必须有来源，不编造数字 ## 何时响应 - 收到 @选品Agent 时必须响应 - 主管指派任务时响应 - 其他情况保持沉默
7

启动并验证

启动所有 Agent，执行存活测试验证部署是否成功

# 启动 gateway openclaw gateway start # 在 Discord 发送存活测试 @主管 测试所有 agent 是否正常工作 # 预期结果：主管 @ 全员，5个Agent在10-40秒内回复"在" 📋 存活测试结果 — 5/5 全员在线 ✅

最佳实践

避坑指南 & 最佳实践

来自真实运营经验的踩坑总结

⚡
主管必须监听全群
主管设置 requireMention: false，这样它能感知整个频道的讨论，适时介入协调。其余专员设为 true，避免噪音干扰。
🧠
主管用高端模型
主管负责决策和协调，值得用 Opus 级别模型。专员执行具体任务，Sonnet 就够了。整体成本降低 80% 同时保持质量。
📝
SOUL.md 要具体
每个 Agent 的 SOUL.md 必须明确：它的专业范围、何时响应、何时沉默。角色越清晰，协作越顺畅，越不会抢戏。
🔧
并行任务用 sessions_yield
spawn 多个子任务后必须调用 sessions_yield()，否则结果只在子 agent 历史里，用户永远看不到。这是多 Agent 协作的核心机制。
🛡️
重要操作人工确认
发邮件、修改线上内容、重大预算决策，Agent 必须请示人工确认。自动化≠无人工干预，关键节点要把关。
📊
定期存活测试
建议每周做一次全员存活测试。让主管 @ 全员，记录响应时间。发现异常（>60秒不回）及时检查 gateway 状态。
